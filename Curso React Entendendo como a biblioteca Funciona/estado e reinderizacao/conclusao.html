[00:00] Parabéns para você que chegou até aqui nesse primeiro curso de react. A primeira vez que a gente está olhando para o framework a gente já viu bastante coisa. Apesar de ser uma aplicação simples, que não tem tanta coisa, não é tão bonita, obviamente que você vai deixar muito melhor colocando

mais funcionalidades, mas já deu para ver como a lib funciona.

[00:20] A gente viu no começo como criar uma aplicação, como instalar o react, depende do node para isso, vimos algumas maneiras de usar o nvm para gerenciar as versões. E a partir daí, uma vez que tínhamos a aplicação criada, começamos a desenvolver ela e entender que o react é uma lib criada pelo Facebook para fazer visualizações dinâmicas, com bastante interação.

[00:46] A gente começou simples, entendendo o que são componentes, como separar os componentes, o que é o jsx. Dentro do projeto, do app, a gente também criou, vimos que é uma classe, começamos com o vclass components, a partir do momento que é uma classe temos o construtor, para ele temos o estado, vimos no finalzinho como renderizar dinamicamente o estado dos objetos e faz com que a visualização seja realmente dinâmica, porque até a gente atualizar não estava, uma vez estava renderizada como estática.

[01:26] A gente começou a fazer nosso projeto e entender os componentes. Vimos que desde o componente fazemos o método render, que é um método gerenciado pelo react, e a partir daí, uma vez que ele é gerenciado pelo react consegue fazer uma série de otimizações através do virtual nome que ele tem.

[01:45] Para funcionar, temos que retornar uma hierarquia do jsx, que é uma extensão do JavaScript que vimos nesse curso. Vimos como trabalhar com ele. Ele deixa a gente definir componentes, lembrando que criamos uma instância desse componente com essa tag, que expresse bastante uma sintaxe de html, de marcação.

[02:12] Lembrando que vimos também que a sintaxe dá mais ou menos a mesma ideia de fazer um new. Toda vez que tenho uma tag, estou dando um new em algum objeto porque cada um desses objetos que criei são uma classe. Se eu navegar até eles podemos ver que temos uma classe. Dentro dessa classe usamos também a parte de propriedades, essa palavra especial, essas props do react, uma palavra especial que serve para fazer injeção de dependências e passar coisas dinamicamente para a classe, na instanciação dela.

[02:43] No caso aqui do formulário usamos as propriedades dela para passar dinamicamente qual o método de criar uma nota que a gente queria que ele usasse. Nessa ideia de criar nota a gente também viu que precisamos lidar com eventos, afinal nosso formulário uma vez que tenho um campo de input, ele vai ser atualizada e quando for atualizado vai disparar no navegador o método de on change.

[03:11] A gente viu que precisamos associar o método da nossa instância, o método desse formulário para o nosso evento de on change daquele objeto de input, e para isso preciso fazer a associação do this, porque vimos que ele é dinâmico no JavaScript, depende de um contexto. Se a gente não fizer o bind, a associação, esse contexto se perde, e se ele se perde a gente tem um erro. É muito comum acontecer, a gente viu nesse curso como associar o bind a um evento que está acontecendo.

[03:41] Bastante coisa que a gente viu. A aplicação ficou simples, mas mostra muito do conteúdo básico do react, e agora você consegue brincar um pouco com isso, criar seus propriedades componentes e fazer a árvore de renderização, de hierarquia que você quiser.

[04:02] Também passamos na nossa lista de notas a parte de JavaScript dinamicamente dentro do jsx de uma maneira que no caso a gente rodasse uma lista, um array. A gente faz um map em cima de um array e a partir dele a gente gera vários elementos que quero que sejam renderizados, porque dessa forma tenho menos código para alterar se precisar alterar alguma coisa. Se precisar alterar a partir dessas notas que foram passadas, se precisar alterar a maneira como estão renderizadas eu altero uma vez e está tudo feito internamente, todas as alterações são replicadas para todas as notas que a gente está criando. É uma vantagem de trazer o JavaScript para junto da renderização.

[04:44] Muita coisa que a gente viu nesse curso, uma introdução bem legal do react. Te espero no próximo curso.
